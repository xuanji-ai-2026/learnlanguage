## 📋 统一接口文件规范

### 后端API接口规范

#### RESTful API设计

**基础URL**: `/api/v1`

**响应格式**:
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

**错误响应**:
```json
{
  "code": 400,
  "message": "错误描述",
  "data": {}
}
```

#### API接口定义

**1. 词汇接口**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/vocabulary/{level} | 获取词汇列表 |
| GET | /api/v1/vocabulary/{id} | 获取词汇详情 |
| POST | /api/v1/vocabulary | 创建词汇 |
| PUT | /api/v1/vocabulary/{id} | 更新词汇 |
| DELETE | /api/v1/vocabulary/{id} | 删除词汇 |

**2. 用户接口**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/v1/users | 创建用户 |
| GET | /api/v1/users/{id} | 获取用户信息 |
| PUT | /api/v1/users/{id} | 更新用户 |
| DELETE | /api/v1/users/{id} | 删除用户 |

**3. 进度接口**

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/v1/progress | 更新学习进度 |
| GET | /api/v1/progress/{user_id} | 获取学习进度 |

### Commit Message格式

**格式**: `<type>(<scope>): <subject>`

**类型（type）**:
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 增加测试
- `chore`: 构建过程变动

**示例**:
```bash
git commit -m "feat(backend): 完成L1级词汇数据"
git commit -m "feat(frontend): 完成词汇列表页面"
git commit -m "fix(vocabulary): 修复词汇加载bug"
```

### TypeScript代码规范

1. **类型定义**: 所有变量必须有类型
2. **接口定义**: 使用interface定义数据结构
3. **错误处理**: 使用try-catch捕获异常

**示例**:
```typescript
import { useState, useMemo } from 'react';

interface Vocabulary {
  id: string;
  wordZh: string;
  wordVi: string;
  pronunciationZh: string;
  pronunciationVi: string;
  meaningZh: string;
  meaningVi: string;
  level: number;
}

const VocabularyCard: React.FC<{ vocabulary: Vocabulary }> = ({ vocabulary }) => {
  const [showBack, setShowBack] = useState(false);

  const handleClick = () => {
    setShowBack(!showBack);
  };

  return (
    <div onClick={handleClick}>
      {showBack ? vocabulary.meaningVi : vocabulary.wordZh}
    </div>
  );
};
```
